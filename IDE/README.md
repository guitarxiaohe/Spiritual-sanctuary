欢迎来到 IDE（Idea Development Environment）文件夹！这里是收集和孵化项目改进想法的地方。

## 📖 目录

- [简介](#简介)

- [规范要求](#规范要求)

## 简介

IDE 文件夹用于记录和讨论关于改进本网站和添加新功能的想法。每个人都可以在这里分享自己的创意和建议，但请确保你的想法有清晰的思路和实现路径。

## 如何使用

### 1. 创建新的想法文件夹

为你的想法创建一个新的文件夹，文件夹名称应该：

- 使用英文小写字母
- 使用连字符 `-` 分隔单词
- 简短且能概括想法的核心内容
- 避免使用特殊字符

**示例**：

```
add-dark-mode/          ✅ 好的命名
Add_Dark_Mode/          ❌ 避免大写和下划线
新功能/                 ❌ 避免中文
add-a-really-long-feature-name-that-describes-everything/  ❌ 避免过长
```

### 2. 创建想法文档

在每个想法文件夹中，创建一个 `README.md` 文件来详细描述你的想法。

## 规范要求

### ✅ 必须包含的内容

每个想法文档必须包含以下部分：

1. **标题和简介**
   - 清晰的标题
   - 简短的功能描述（2-3 句话）

2. **背景和动机**
   - 为什么需要这个功能？
   - 解决什么问题？

3. **详细设计**
   - 功能描述
   - 用户界面设计（如有）
   - 技术实现方案
   - 数据结构设计（如需要）

4. **实现步骤**
   - 分步骤的实现计划
   - 每个步骤的预期结果

5. **技术考虑**
   - 需要使用的技术栈
   - 可能遇到的技术难点
   - 性能影响评估

### 📋 建议包含的内容

- **替代方案**：考虑过的其他实现方式
- **参考资料**：相关的文档、链接、截图等

### ⚠️ 注意事项

1. **思路清晰**：确保你的想法有明确的实现路径，不仅仅是"我想要这个功能"
2. **可行性**：考虑技术可行性和资源限制
3. **影响范围**：说明这个功能对现有系统的影响

## 文件夹结构

```
IDE/
├── README.md                    # 本文件
├── add-dark-mode/              # 示例：添加深色模式
│   ├── README.md               # 想法详细描述
│   ├── mockup.png              # 设计稿（可选）
│   └── references.md           # 参考资料（可选）
├── improve-search/             # 示例：改进搜索功能
│   └── README.md
└── your-idea-name/             # 你的想法
    └── README.md
```

## 🤝 贡献指南

1. **搜索现有想法**：在创建新想法前，先查看是否已有类似的建议
2. **遵循规范**：按照模板和规范编写文档
3. **清晰表达**：使用简洁明了的语言
4. **提供细节**：尽可能提供详细的设计和实现方案
5. **保持开放**：欢迎他人的反馈和建议
